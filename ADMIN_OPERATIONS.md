# 管理端操作说明：加新店与装修

## 一、加新店铺（新域名、同一套下单审核）

1. 登录管理后台，进入 **店铺管理（Shop Management）**。
2. 点击 **新建店铺**。
3. **源码选择**：当前仅有「胡须店」；日后有服装店、手机店等源码时会在此列出，选对应类型即可。
4. 填写 **店铺名称**、**主域名**（如 `clothing.example.com`），点击 **创建店铺**。
5. 将该域名的 DNS A 记录指向当前服务器；如需更多域名，在该店铺卡片下「绑定域名」处添加并保存。
6. 该域名访问到的前台页面、商品由所选「源码」决定；**下单与审核**走同一套后端，订单在「数据列表」中按店铺区分。

**说明**：同一台服务器可挂多个店铺（多域名），每个店铺可选不同源码；订单统一在管理端审核，客户填写的信息（姓名、地址等）原样展示，不随管理端语言切换而改变。

---

## 二、装修现有店铺（改页面、板块、图片）

**当前能力**：  
- 店铺的**名称、主域名、绑定域名、IP 防护策略**等，均在管理端「店铺管理」中可改。  
- **页面样式、板块、商品、图片**目前写在前端代码与静态资源里，**无法在管理后台里直接装修**。

**可选做法**：

1. **改代码再部署**  
   在电脑上修改对应前端（如胡须店源码里的组件、`public/` 图片、商品数据等），然后 `git push`，在服务器上 `git pull` 并执行 `bash update.sh` 重新构建部署。适合一次性大改或新店用新源码。

2. **日后做「后台装修」**  
   若需要：在管理端选择店铺 → 编辑首页板块、上传图片、配置商品等，并保存到数据库，前台按配置渲染。这需要开发「店铺装修」功能（后端存配置 + 管理端编辑页 + 前台读配置渲染），可按需求再加。

---

## 三、多店铺与审核

- **多店铺**：在「店铺管理」中可创建多个店铺，每个店铺可有多个域名；每个店铺可选一个「源码」类型（当前仅胡须店）。
- **审核**：所有店铺的订单都在 **数据列表** 中展示，可筛选、按订单通过/拒绝/退回优惠券/确认短信等；**客户下单时填写的所有信息均按用户填写原样显示，不随管理端中/英文切换而改变**。
